AMF Identifier

The AMF Identifier allows you to perform brute force identification of destination and methods using a dictionary file.

The basic premise of the tool is that you specify the AMF URI of the service you want to enumerate. If you are enumerating methods, then you must also provide the destination name. Once all the options are selected, the AMF Identifier will start querying the service and inspecting error messages to determine if the destination or method exists on the server.

RO Endpoint:

The URL of the AMF Gateway that has the function you are calling.

RO Destination:

The destination name that is to supplied to the RO Endpoint.

Brute Force:

Select whether you are trying to brute force destinations or methods.

Auth Information:

The username and password for the connection. The Remote Credentials checkbox dictates whether the RemoteObject.setCredentials or the RemoteObject.setRemoteCredentials function is used to set authentication information on the connection.

Dictionary File:

This will create a pop-up where you can select the dictionary file. It will only show files ending in .txt or .dict. Doubel click on the file you want to use in order to select. You can also manually enter the location of the file using the native path for the OS. e.g. C:\test.txt

Query Interval:

The AMF Identifer will make a request every 330 milliseconds by default. Adjust this value to adjust the number of queries per second.

Start Queries:

The Start Queries button will begin attempting to enumerate services.

Reset:

This will stop any running queries and reset everything back to the beginning. You will need to click Start Queries to restart the scan.

Output:

This will contain the response from the remote query. Anytime a destination or service is found, it will be placed at the top of the Output box and start with the word FOUND followed by additional information. All failure information is tagged onto the end of the output as responses come back.
