AMF Fuzzer

The AMF Fuzzer is based on the AMF Transmitter and it allows you to query remote AMF Services and retrieve the result. This feature is still experimental until a way is determined to dynamically set Remote alias information for an object.

The basic premise of the tool is that you specify the AMF URI, the destination name and the function you wish to call in the first three fields. Then you add any parameters that should be sent. You can set any creds that need to be sent with the connection. These should all be the baseline values for a normal request. Then, select which fields you want to fuzz by setting the "fuzz" field to "y". You can specify custom fuzz parameters using a fuzz.xml

RO Endpoint:

The URL of the AMF Gateway that has the function you are calling.

RO Destination:

The destination name that is to supplied to the RO Endpoint.

RO Function:

The name of the remote function that you will call.

Function Parameters/New Value:

This is where you add the parameters that will be sent across the AMF connection. This is similar to the functionality in the AS3 Inspectors. For information on how to add Classes and Objects, please click here.

The text field at the top should include the class name that you would supply to a registerClassAlias call in ActionScript

Auth Information:

The username and password for the connection. The Remote Credentials checkbox dictates whether the RemoteObject.setCredentials or the RemoteObject.setRemoteCredentials function is used to set authentication information on the connection.

Submit Query:

The Submit Query button executes the call to the remote service.

Output Tab:

This will contain the response from each query.

Save Response:

This button allows you to save the response log.
