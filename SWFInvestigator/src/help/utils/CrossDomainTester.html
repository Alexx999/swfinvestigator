Cross-Domain Tester

The cross-domain tester allows you to test whether your cross-domain policy file will allow loading from a specific domain. Enter the domain that you want to attempt to load data from in the text box at the top of the page and click Re-Map. This will load the Cross-Domain Tester in the security context of that domain. Then, provide the relevant information for the load operation in the fields below and click Submit Request to determine if the cross-domain policy on the target site will allow the data to load.

Important Note

Cross-domain policy files are cached by Flash Player. If you want to test a new version of the cross-domain policy file, then click Re-Map at the top of the window to reload the SWF and clear the cache.

URL

This is the URL for the data that you want to load. It should be the site that hosts the cross-domain policy file.

METHOD

This is the HTTP request method for this request ("GET","POST", etc.).

POST BODY

If this request is a POST, then you would provide the data that you would send with the post request.

HTTP Header Checkbox

If this request will send custom HTTP headers, then enable this check box. Custom HTTP headers typically have to be allowed by the cross-domain policy file.

HTTP Header

Within these two text boxes, you would provide the header information. The first text box is the header name. The second text box is the header value.

Policy File

If you need to load a cross-domain policy file from a sub-directory, then use this text box to provide the URL of the child cross-domain policy file. Policy files in sub-directories must be supported within the master cross-domain policy file.

Submit Request

Once all of the information is filled out for this cross-domain request, pressing this button will submit the request to the remote server. The response will display in the Output text box to the right.

Output

The result from the cross-domain request will display in this text box. You can also confirm that the domain that you have chosen at the top of the page has taken effect. When the SWF first loads, it will display its current domain security context within this text field.
