Object Editor

The Object Editor allows you edit ActionScript objects using definitions in memory, from dynamically loaded SWFs, saved AMF files or by custom crafting them.

Class Info

This will contain a text representation of the object that is created from the class. The Register Class Name and Save To File button will allow you to create a local AMF file for the object. By checking Register Class Name, the ObjectEditor will try to register the class name in the Fetch Class section with the object before saving so that it isn't lost when the object is converted to AMF.

Read File

Click this button if you have already have an AMF file with the object defined that you can load from the filesystem. This is an alternative to the Fetch Class approach.

Fetch Class

If you want to create an object dynamically, then type in the Fully Qualified Name of the class (eg. mx.controls.TextArea). Next you will tell the object editor where to find the class. Your options are:

  • System - A builtin object or one the Object Editor already knows about.
  • Remote - Import load (via Loader.loadBytes) a SWF that has the class.
  • Local - Import load a SWF from the file system that has the class.

You may see an error when loading the SWF but you can probably ignore it. Click Get class definition to fetch the class.

Variable to Assign

This is the same variable interface that is used throughout SWF Investigator.

Debug Output

This is just left over from debugging the implementation. Debug information can be found in this display.
